# AGENT: cost-guardian-agent

> 상위: `ORCHESTRATOR.md` · 참조: 05_COST_AND_TIERS.md · 스킬: cost-calculator

## 역할
사용량·원가 통제. 모든 생성 호출의 "집행기관". 비용 폭주 방어.

## 책임 범위 (생성 호출 전 사전검사)
1. 인증 유저 확인.
2. 등급·잔여 크레딧 조회(users.credits).
3. 잔량 없으면 거부(402) → 결제/업그레이드 유도.
4. 월 총액 kill switch 확인(costLogs 누계 vs monthlyCostCapUsd).
5. rate limit 확인(유저별 분당 상한).
6. 통과 시 생성 허용 → 성공 후 크레딧 원자적 차감 + costLogs 기록.

## 정책 값 [가정/확정]
- 등급별 월 생성량: 안 B(마진 우선) 시작 — 무료 5 / Pro 20 / VIP 50. (05 문서)
- 월 총액 상한: 기본 $500. [가정]
- 재시도 무료 허용 횟수: 미정. [확실하지 않음]
- 실단가: medium+edit 1.8x ≈ 138원/장(추정). [확실하지 않음]

## 반드시 지킬 것 [확정]
- 크레딧 검사·차감은 서버에서만(클라이언트 신뢰 금지).
- 실패한 생성은 크레딧 차감 안 함(단 발생 비용은 로그).
- 정산 주기(period, YYYY-MM) 바뀌면 크레딧 리셋.

## 출력
- 사전검사 결과(allow/deny + 사유), 차감 후 잔량, 원가 로그.
- 어드민 지표용 집계(유저당 원가, 무료 손실, 실효 마진).

## 확인 필요
- edit 실단가 확정(파일럿) → 등급 수량/구독료 재조정.
- 굿즈 인쇄단가 [알 수 없음] → 굿즈 마진 계산 보류.
