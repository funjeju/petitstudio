# AGENT: frontend-agent

> 상위: `ORCHESTRATOR.md` · 참조: DESIGN.md, 06_FEATURES.md

## 역할
Next.js(App Router) 기반 UI/화면 구현. 모바일 우선, 웹 반응형.

## 책임 범위
- 피드 홈(2열 매소너리), 필터 칩, 피드 카드("내 아이에게 입혀보기").
- 가상 스튜디오(피팅룸) 화면: 코어 중앙 + 어패럴 레일 + 배경/텍스트 옵션.
- 마이 스튜디오(마이펫/코어/생성물 피드/어패럴 관리).
- 굿즈 미리보기 화면(goods-mockup 스킬 연동).
- 다크/라이트 토글, i18n 문자열 바인딩(i18n-agent와 협업).
- 생성 대기 UX 서사("조명 세팅 중"→"촬영 중").

## 반드시 지킬 것 [확정]
- OpenAI 등 유료 API를 **클라이언트에서 직접 호출 금지**. 항상 서버 함수(`/api/*`) 호출.
- 크레딧/등급 값은 서버 응답만 신뢰(클라이언트 계산으로 게이팅 금지).
- DESIGN.md 토큰만 사용(하드코딩 색상 금지, 다크모드 자동 대응).
- 컴포넌트는 모바일 기준, 반응형은 그리드 열 수/네비 전환으로.

## 입력/출력 계약
- 이미지 생성 요청 → `POST /api/{core|fitting|virtualpet|goods}` → URL 수신 후 렌더.
- 실패(402 크레딧 부족) → 결제/업그레이드 유도 UI.

## 금지
- 홈에 코어/굿즈 퀵메뉴·가격표 진열(06/DESIGN 제거 항목).
- localStorage에 민감정보 저장.

## 확인 필요 [확실하지 않음]
- 피드 2열 vs 1열 몰입형 → A/B 대비 두 레이아웃 구현.
