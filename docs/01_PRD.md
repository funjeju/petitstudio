# 01. 제품 요구사항 정의 (PRD)

> 상위: `ORCHESTRATOR.md` · 표기 규칙 동일([확정]/[가정]/[확실하지 않음]/[알 수 없음])

---

## 1. 문제 정의

- 반려동물 사진관 촬영은 비싸고 번거롭다(반려동물 특성상 스튜디오 촬영 난이도 높음).
- 기존 AI 펫 프로필 서비스는 **1회성 이미지 판매**에 그쳐 재구매·리텐션이 약하다.
  - 근거: 헬로우봇 '반려동물 AI 프로필'은 수요 검증에는 성공했으나 운영사 띵스플로우는 자본잠식 후 2025-05 폐업, 헬로우봇은 매각됨. [확정]
    - 출처: 유니콘팩토리 https://www.unicornfactory.co.kr/article/2025010713491094660 , THE VC https://thevc.kr/thingsflow
- 생성된 근사한 AI 이미지를 **실물 굿즈로 연결하는 완결형 서비스**는 국내에서 확인되지 않음. [확실하지 않음 — 검색 범위 내 판단]

## 2. 목표

- 코어 이미지를 자산화해 재방문·재구매를 유도.
- '입혀보기' 놀이를 굿즈 구매로 전환.
- 연말 계절성 극복(입양일/생일 기준 시작월 상품, 이벤트 기점 굿즈).

## 3. 타깃 사용자 [가정]

1차: 반려동물을 가족으로 기록·공유하고 싶은 반려인(20~40대).
2차: 반려동물이 없어도 '가상 펫 만들기'로 유입되는 창작·수집·바이럴 유저.
→ 정확한 세그먼트 비중은 출시 후 데이터로 확정. [알 수 없음]

## 4. 핵심 사용자 여정

```
[유입] 피드에서 남의 착장 구경
   → "내 아이에게 입혀보기" 클릭
[온보딩] 사진 3장 업로드 → 코어 이미지 생성 (첫 무료 체험)
[핵심] 가상 피팅룸: 어패럴/액세서리 장착 + 배경 선택 → 착장 이미지 생성(자동 저장)
[수집] 마음에 든 착장을 컬렉션에 저장
[전환] 컬렉션 12벌 → 달력 만들기 / 단일 착장 → 액자·폰케이스
[공유] SNS 퍼가기 → 다시 피드로 유입 루프
```

## 5. 기능 범위 (MVP)

### 반드시 (Must)
- Google 로그인, 등급 체계(비회원/무료/Pro/VIP).
- 코어 이미지 생성(실제 펫 사진 업로드).
- 가상 피팅룸(어패럴·액세서리 장착, 배경 옵션 + 유저 텍스트).
- 생성 이미지 자동 저장 + 소프트 삭제.
- 마이 스튜디오(마이펫/코어/생성물 피드/어패럴 관리).
- 굿즈 미리보기 + 생성원가 계산(주문은 목업 단계 가능).
- 어드민(회원·데이터 관리, 품질 티어 선택, 지표 로그).
- 다국어(18개), 다크/라이트, SNS 공유.

### 하면 좋음 (Should)
- 가상 펫 생성(2~3개 동물/전설 캐릭터 샘플 + 프롬프트 합성).
- 피드 팔로우/좋아요/저장.

### 이후 (Could / 보류)
- 실물 인쇄 제휴 주문 완결 — **[알 수 없음] 제휴 단가 확정 후.**
- 펫용품 커머스 제휴.

## 6. 비기능 요구사항

- **비용 안전**: 이미지 생성은 서버 함수 경유 + 등급별 사전 차감 + 월 총액 kill switch.
- **자산 보존**: 생성 이미지는 비용이 발생한 자산이므로 자동 저장, 유저 명시 삭제 전 보존.
- **성능**: 생성 대기 시간(수 초~수십 초)을 UX 서사로 전환("촬영 중", "조명 세팅 중").
- **반응형**: 모바일 우선, 웹은 동일 컴포넌트로 자동 대응(브레이크포인트는 DESIGN.md).

## 7. 성공 지표 (KPI) [가정]

- 온보딩 완료율(가입→코어 이미지 1장 생성).
- 무료→Pro 전환율.
- 유저당 월 생성 수 / 유저당 원가.
- 컬렉션→굿즈 전환율.
- 재방문율(코어 자산 보유 유저의 W2/W4 리텐션).
