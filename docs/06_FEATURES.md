## 1. 화면 구조 (IA)

### 모바일 (5탭 하단바)
```
홈(피드) | 탐색 | [중앙 생성 FAB] | 컬렉션 | 마이 스튜디오
```
- 웹에서는 하단바 → 좌측 사이드바로 전환(02 반응형 전략).

### 홈 = 피드 (주인공은 유저 결과물)
- Zone1 헤더: 워드마크 + 검색/알림.
- Zone2 개인화 스트립: 상태별 1줄 카드(신규="사진 3장이면 초코의 스튜디오 컷", 보유="가을 신상 도착").
- Zone3 필터 칩: 추천 / 팔로잉 / 시즌 / 스타일 태그 / **실험실(가상펫)**.
- Zone4 피드: 2열 매소너리. 카드 = 착장 이미지 + 펫이름·품종 + 아이템 칩 + **"내 아이에게 입혀보기"** 버튼 + 좋아요/저장.
- 중앙 FAB: 코어 없으면 촬영(업로드)→코어 생성, 있으면 바로 피팅룸.

> 원 시안(첨부 4개)의 "코어 이미지/가상 피팅룸/굿즈/주문" 4버튼 퀵메뉴는 **제거**.
> 코어 이미지는 홈 메뉴가 아니라 **피팅룸 진입 시 생성되는 상태**이자 마이 스튜디오 자산.

## 2. 가상 스튜디오 (피팅룸) — 핵심 화면

- 중앙: 코어 이미지(스튜디오 분위기).
- 하단/사이드: 어패럴·액세서리 레일(모자/의상/액세서리/시즌/소품 카테고리).
- 장착 규칙: anchor 기반 자연스러운 합성(04 문서, image-edit-orchestration 스킬).
- 배경 선택: 바다/숲/도시/거실 프리셋 + 유저 텍스트.
- 각 장착/변경 시 medium으로 미리보기 생성(자동 저장).
- "달력용으로 12벌 채우기" 등 컬렉션 유도.

## 3. 마이 스튜디오 (마이페이지)

- **마이펫 관리**: 펫 등록/수정(이름·품종·생일·입양일), 실제/가상 구분.
- **코어 이미지 관리**: 펫별 코어 목록, 재생성, 대표 코어 지정.
- **생성 이미지 피드 관리**: 내 착장/합성 결과를 피드 형식으로 열람·정렬·삭제(소프트).
- **장착 어패럴 관리**: 즐겨찾는 어패럴, 저장한 착장 조합.
- 주문 내역, 설정(등급·언어·테마).

## 4. 가상 펫 생성

- 2~3개 동물/전설 캐릭터 샘플 업로드 + 프롬프트 설명.
- 생성 후 코어化 → 기존 파이프라인 재사용.
- 실험실 탭/필터에서 진입, 피드 바이럴 엔진.
- 정책: 실존 인물/저작권 IP 합성 금지(04 §4).

## 5. 굿즈 전환

- 컬렉션 → "굿즈로 만들기" → 굿즈 종류 선택 → 미리보기(goods-mockup 스킬).
- 굿즈 확정 시 high 재생성(생성원가 발생, 05 문서).
- 달력은 입양일/생일 기준 시작월 지원(계절성 극복).
- 주문 완결(결제/배송)은 인쇄 제휴 확정 후. [알 수 없음]

## 6. 자동 저장 & 소프트 삭제 [확정]

- 모든 생성 이미지(코어 포함) 생성 즉시 자동 저장.
- 삭제는 유저가 명시적으로 삭제 버튼을 눌러야만 → `status: trashed` → 유예 후 하드 삭제.
- 비용이 든 자산이므로 시스템이 임의 삭제하지 않음.

## 7. SNS 공유 [확정]

- 모든 생성물(코어 포함)에 SNS 퍼가기 버튼.
- 공유 이미지에 서비스 워터마크/핸들 옵션(바이럴 + 브랜딩). [가정]
- 피드 공개는 옵트인(shared 플래그). 기본 비공개 vs 공개는 A/B. [확실하지 않음]

## 8. 다국어 (i18n) [확정]

- 18개 언어 지원. 상세 목록·구현은 i18n-agent 문서.
- 모든 UI 문자열 키화, 어패럴/굿즈 명칭은 `nameI18n` 맵(03 문서).
- 로케일 자동 감지 + 유저 설정 저장(users.locale).

## 9. 다크/라이트 테마 [확정]

- 토글 스위치(시스템/라이트/다크). users.theme 저장.
- 디자인 토큰은 DESIGN.md(시안 3번 기준: 화이트+그레이, 다크 변형 정의).

## 10. 접근성/성능 [가정]

- 생성 대기 시간을 스튜디오 서사로 연출("조명 세팅 중"→"촬영 중").
- 이미지 지연 로딩, 피드 무한 스크롤.
- 콜드스타트: 운영 시드 룩북으로 피드 초기 채움.
