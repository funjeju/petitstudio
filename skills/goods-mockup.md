# SKILL: goods-mockup

> 사용 주체: frontend-agent, cost-guardian-agent · 참조: 05_COST_AND_TIERS.md, 06_FEATURES.md
> 목적: 생성 이미지를 굿즈에 얹어 미리보기 + 굿즈별 원가 계산.

## 언제 쓰나
- 컬렉션/착장 → "굿즈로 만들기" 진입 시.

## 굿즈 타입과 요건
| 굿즈 | 필요 이미지 | 권장 비율 | 생성원가(high) |
|---|---|---|---|
| 탁상 달력 | 12장 | 세로 1024×1536 | 약 2,868원 |
| 벽걸이 달력 | 12장 | 세로 | 약 2,868원 |
| 액자 | 1장 | 세로/정사각 | 약 239~306원 |
| 폰케이스 | 1장 | 세로 | 약 239원 |

- 생성원가 = HIGH 단가 × 필요 이미지 수(05/cost-calculator).
- **인쇄 제휴원가는 [알 수 없음]** — 국내 인쇄사 실견적 전 굿즈 마진 확정 불가.

## 달력 특수 규칙 [확정]
- 시작월: 1월 고정 외에 **입양일/생일 기준 시작월** 지원(계절성 극복).
- 12벌 착장 컬렉션 → 12개월 매핑 UI.

## 미리보기 처리
- 미리보기는 medium 이미지를 목업 프레임(탁상/벽걸이/액자/케이스)에 합성해 표시.
- **주문 확정 시 high로 재생성**(인쇄 해상도) → 생성원가 발생.
- 목업은 실제 상품 맥락(책상 위 등)으로 보여 소유 상상 자극.

## 원가 표시 로직
```
goods_total_cost = gen_cost(high, required_images) + print_cost(?) + fee + shipping
# print_cost = [알 수 없음] → 확정 전 "인쇄단가 미정" 표기
```

## 확인 필요
- 인쇄 해상도 요건(2K/4K, 업스케일 필요 여부) → 제휴사 확정 후 [알 수 없음].
- 굿즈 판매가는 현재 [가정]. 인쇄단가 확정 후 마진 역산.
